GITHUB PAGES

Github Pages är skapat för att publicera direkt vid versionshantering. Vid commits/push pbliceras sidan direkt vilket inte kräver FTP/SFTP. Jag har tidigare inte använt sådan publicering men FileZilla mot MIUN-servr.

VAD ÄR GITHUB PAGES

Github är en versionshanteringslösning för alla som skapar kodprojekt. Genom Github Pages kan publikation av statiska webbplatser ske. Med en repo med användarnamn + github.io. kan webbplatsen skapas Genom "Pages" vilket är en gratis tjänst.

FÖRDEL/NACKDEL

Fördelar med Github Pages är många. Den är gratis tjänst som är även enkel att använda med eller utan erfarenhet av Git-versionshantering. Det tog några minuter att skapa ett repo på rätt sätt och klona det lokalt för att skapa webbfiler. Med en enda commit/push så är webbplatsen publicerad med Tailwind och Bootstrap. Github Pages fungerar med HTML/CSS/JS. Vidare kan flera kodare göra commits/push till en enda webbplats som är live, för att sedan merga till main branch - olika utvecklare kan arbeta med olika delar av webbplatsen och uppdatera den dynamiskt. Det är snabbare att visa ändringar än FTP-lösning som FileZilla. Webbplatsen kan även versionhanteras dynamiskt.

Nackdelar inkluderar att det inte finns stöd för PHP-server eller backend-utveckling samt inte heller SQL-databaser eller andra databaser. Det tar cirka 10-20 sekunder för att en commit/push blir live på webbplatsen till skillnad från Live-server. Tidsskillnader gör att små ändringar måste pushas varje gång för resultat vilket är tidskrävande och frustrerande.
